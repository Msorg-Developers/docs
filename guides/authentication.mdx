## Overview

The Dancity API uses **Bearer token authentication**. Every request must include your API key in the `Authorization` header, plus the mandatory `channel` header that identifies where the request originates.

```http
Authorization: Bearer dcy_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
channel: API
```

<Warning>
  Your API key grants full access to your Dancity account. **Never expose it in
  client-side code, public repositories, or logs.** Store it only in server-side
  environment variables.
</Warning>

## API Key format

All Dancity live keys follow the format:

```
dcy_live_<random-40-char-hex>
```

The key is shown **only once** immediately after generation. If you lose it, you must regenerate — the old key is immediately invalidated.

## Sending the Authorization header

<CodeGroup>
```bash cURL
curl https://api.dancity.app/wallet \
  -H "Authorization: Bearer dcy_live_xxxx..." \
  -H "channel: API"
```

```javascript Node.js / fetch
fetch("https://api.dancity.app/wallet", {
  headers: {
    Authorization: "Bearer dcy_live_xxxx...",
    channel: "API",
  },
});
```

```python Python / requests
import requests

requests.get(
    "https://api.dancity.app/wallet",
    headers={
        "Authorization": "Bearer dcy_live_xxxx...",
        "channel": "API",
    },
)
```

```php PHP
$ch = curl_init("https://api.dancity.app/wallet");
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    "Authorization: Bearer dcy_live_xxxx...",
    "channel: API",
]);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$response = curl_exec($ch);
```
</CodeGroup>

## The `channel` header

The `channel` header is **required** on all requests and tells Dancity the source of the request. For API integrations, always send:

```
channel: API
```

| Value | Description |
|-------|-------------|
| `API` | Server-to-server integration |
| `WEB` | Web browser |
| `IOS APP` | iOS mobile app |
| `ANDROID APP` | Android mobile app |
| `USSD` | USSD channel |

## IP Allowlist

For additional security, you can restrict your API key to specific IP addresses or CIDR ranges from your [profile page](https://dancity.app/profile/api-key). This requires an OTP verification when adding or changing IPs.

<Info>
  **An empty IP allowlist blocks all traffic.** You must add at least one IP or
  CIDR range after enabling the allowlist.
</Info>

**Supported formats:**

| Format | Example |
|--------|---------|
| IPv4 single address | `203.0.113.5` |
| IPv6 single address | `2001:db8::1` |
| IPv4 CIDR range | `192.168.1.0/24` |
| IPv6 CIDR range | `2001:db8::/32` |

Up to **20 entries** are allowed.

## Unauthorized responses

If authentication fails, the API returns:

```json
{
  "statusCode": 401,
  "message": "Unauthorized"
}
```

Common causes:
- Missing or malformed `Authorization` header
- Revoked or invalid API key
- Request originated from an IP not in your allowlist
- Your API client has been deactivated

## Key rotation

To rotate your API key:

1. Go to your [API Key page](https://dancity.app/profile/api-key)
2. Click **Regenerate API Key**
3. Enter the OTP sent to your email
4. Copy and store the new key immediately
5. Update all your integrations — the old key stops working instantly
