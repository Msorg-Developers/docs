## Welcome

The Dancity HTTP API covers merchant VAS (`/api/external/v1`), in-app VAS and wallets (`/api/...` with a JWT), catalog (services, products, plans), eSIM, virtual cards, education pins, and betting. The **API reference** tab lists **one page per operation** (like PayVessel): each page pulls **Try it** and schemas from the shared [OpenAPI](https://www.openapis.org/) spec.

## Before you start

<Steps>
  <Step title="Dancity account (Tier 3)">
    Sign up at [dancity.app](https://dancity.app). API access requires Tier 3 and an approved API client.
  </Step>
  <Step title="API key">
    After approval, generate a merchant key from the API Key page. **Copy once** — it is not shown again.
  </Step>
  <Step title="Allowlist IPs / webhook">
    Add server IPs and (optionally) an HTTPS `webhookUrl` for server-to-server events.
  </Step>
  <Step title="Integrate">
    Open the [API reference](/api-reference/introduction) tab and the group for your product (e.g. **VAS** or **Wallet**). Each page matches one HTTP operation and **Try it**.
  </Step>
</Steps>
