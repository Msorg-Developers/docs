## Overview

The transactions endpoints let you look up your transaction history, verify the status of a specific transaction, and retrieve summary statistics.

---

## List Transactions

Returns a paginated list of transactions for your account, with optional filters.

```http
GET /transactions
```

### Headers

| Header | Value | Required |
|--------|-------|----------|
| `Authorization` | `Bearer <api-key>` | ✅ |
| `channel` | `API` | ✅ |

### Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `page` | number | Page number (default: `1`) |
| `limit` | number | Results per page (default: `20`, max: `100`) |
| `type` | string | Filter by type: `AIRTIME`, `DATA`, `ELECTRICITY`, `CABLE`, `TRANSFER`, `FUNDING` |
| `status` | string | Filter by status: `success`, `failed`, `pending` |
| `startDate` | string | ISO 8601 date — filter from this date |
| `endDate` | string | ISO 8601 date — filter up to this date |
| `reference` | string | Search by your `customerRef` |

### Example request

<CodeGroup>
```bash cURL
curl "https://api.dancity.app/transactions?type=AIRTIME&status=success&page=1&limit=10" \
  -H "Authorization: Bearer dcy_live_xxxx..." \
  -H "channel: API"
```

```javascript Node.js
const params = new URLSearchParams({
  type: "AIRTIME",
  status: "success",
  page: "1",
  limit: "10",
});
const res = await fetch(`https://api.dancity.app/transactions?${params}`, {
  headers: {
    Authorization: `Bearer ${process.env.DANCITY_API_KEY}`,
    channel: "API",
  },
});
const data = await res.json();
```

```python Python
import requests, os

resp = requests.get(
    "https://api.dancity.app/transactions",
    params={"type": "AIRTIME", "status": "success", "page": 1, "limit": 10},
    headers={
        "Authorization": f"Bearer {os.environ['DANCITY_API_KEY']}",
        "channel": "API",
    },
)
```
</CodeGroup>

## Get Transaction by ID

Retrieve a single transaction by its ID. Use this to verify the status of a specific purchase.

```http
GET /transactions/:transactionId
```

### Path parameters

| Parameter | Description |
|-----------|-------------|
| `transactionId` | The transaction's `_id` or `transactionId` |

### Example request

<CodeGroup>
```bash cURL
curl https://api.dancity.app/transactions/TXN-2024-00123 \
  -H "Authorization: Bearer dcy_live_xxxx..." \
  -H "channel: API"
```

```javascript Node.js
const res = await fetch("https://api.dancity.app/transactions/TXN-2024-00123", {
  headers: {
    Authorization: `Bearer ${process.env.DANCITY_API_KEY}`,
    channel: "API",
  },
});
const data = await res.json();
```
</CodeGroup>

## Transaction Statistics

Get an aggregate summary of your transaction activity.

```http
GET /transactions/stats/summary
```

### Example request

<CodeGroup>
```bash cURL
curl https://api.dancity.app/transactions/stats/summary \
  -H "Authorization: Bearer dcy_live_xxxx..." \
  -H "channel: API"
```
</CodeGroup>

### Response

```json
{
  "success": true,
  "message": "Transaction statistics",
  "data": {
    "totalTransactions": 158,
    "totalSpent": 85200.00,

```

## Transaction statuses

| Status | Meaning |
|--------|---------|
| `success` | Transaction completed and service delivered |
| `pending` | Transaction is being processed — poll again shortly |
| `failed` | Transaction did not complete — wallet was not debited |

<Warning>
  A `pending` status does not mean the service was not delivered. Always wait
  for the status to resolve to `success` or `failed` before retrying. Use
  [webhooks](/guides/webhook) to be notified instantly when a status changes.
</Warning>
