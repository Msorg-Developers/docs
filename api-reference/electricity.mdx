## Overview

Pay electricity bills for all Nigerian distribution companies (DISCOs). The flow is:

1. **Validate** the meter number → get customer name and meter type
2. **Purchase** → vend the token or post a payment

---

## Validate Meter

Verify a meter number and retrieve customer details before initiating a purchase.

```http
GET /electricity/validate
```

### Headers

| Header | Value | Required |
|--------|-------|----------|
| `Authorization` | `Bearer <api-key>` | ✅ |
| `channel` | `API` | ✅ |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `meterNumber` | string | ✅ | The meter number to validate |
| `product` | string | ✅ | DISCO product name (e.g. `"IKEDC Prepaid"`) |

### Example request

<CodeGroup>
```bash cURL
curl "https://api.dancity.app/electricity/validate?meterNumber=12345678901&product=IKEDC%20Prepaid" \
  -H "Authorization: Bearer dcy_live_xxxx..." \
  -H "channel: API"
```

```javascript Node.js
const params = new URLSearchParams({
  meterNumber: "12345678901",
  product: "IKEDC Prepaid",
});
const res = await fetch(`https://api.dancity.app/electricity/validate?${params}`, {
  headers: {
    Authorization: `Bearer ${process.env.DANCITY_API_KEY}`,
    channel: "API",
  },
});
const data = await res.json();
```

```python Python
import requests, os

resp = requests.get(
    "https://api.dancity.app/electricity/validate",
    params={"meterNumber": "12345678901", "product": "IKEDC Prepaid"},
    headers={
        "Authorization": f"Bearer {os.environ['DANCITY_API_KEY']}",
        "channel": "API",
    },
)
```
</CodeGroup>

### Response

```json
{
  "success": true,
  "message": "Meter validated successfully",
  "data": {
    "customerName": "JOHN DOE",
    "meterNumber": "12345678901",
    "address": "12 SAMPLE STREET, IKEJA",
    "meterType": "PREPAID",
    "disco": "IKEDC"
  }
}
```

## Purchase Electricity

Pay the electricity bill and receive a vending token (prepaid) or payment reference (postpaid).

```http
POST /electricity/purchase
```

### Headers

| Header | Value | Required |
|--------|-------|----------|
| `Authorization` | `Bearer <api-key>` | ✅ |
| `Content-Type` | `application/json` | ✅ |
| `channel` | `API` | ✅ |

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `product` | string | ✅ | DISCO product name (e.g. `"IKEDC Prepaid"`) |
| `meterNumber` | string | ✅ | The meter number |
| `amount` | number | ✅ | Amount in NGN. Min: `1000` |
| `phone` | string | ✅ | 11-digit phone number for token delivery |
| `pin` | string | ✅ | Your 4-digit transaction PIN |
| `customerRef` | string | ❌ | Your own reference for idempotency |

### Supported DISCOs

| Product name | Type |
|---|---|
| `IKEDC Prepaid` | Prepaid |
| `IKEDC Postpaid` | Postpaid |
| `EKEDC Prepaid` | Prepaid |
| `EKEDC Postpaid` | Postpaid |
| `AEDC Prepaid` | Prepaid |
| `AEDC Postpaid` | Postpaid |
| `PHED Prepaid` | Prepaid |
| `PHED Postpaid` | Postpaid |
| `KEDCO Prepaid` | Prepaid |
| `BEDC Prepaid` | Prepaid |
| `JED Prepaid` | Prepaid |
| `IBEDC Prepaid` | Prepaid |

### Example request

<CodeGroup>
```bash cURL
curl -X POST https://api.dancity.app/electricity/purchase \
  -H "Authorization: Bearer dcy_live_xxxx..." \
  -H "Content-Type: application/json" \
  -H "channel: API" \
  -d '{
    "product": "IKEDC Prepaid",
    "meterNumber": "12345678901",
    "amount": 5000,
    "phone": "08012345678",
    "pin": "1234",
    "customerRef": "ELEC-ORDER-001"
  }'
```

```javascript Node.js
const res = await fetch("https://api.dancity.app/electricity/purchase", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.DANCITY_API_KEY}`,
    "Content-Type": "application/json",
    channel: "API",
  },
  body: JSON.stringify({
    product: "IKEDC Prepaid",
    meterNumber: "12345678901",
    amount: 5000,
    phone: "08012345678",
    pin: "1234",
    customerRef: "ELEC-ORDER-001",
  }),
});
const data = await res.json();
```
</CodeGroup>

### Response

```json
{
  "success": true,
  "message": "Electricity purchase successful",
  "data": {
    "transactionId": "TXN-2024-00789",
    "reference": "dcy_ref_ghi789",
    "customerRef": "ELEC-ORDER-001",
    "meterNumber": "12345678901",
    "customerName": "JOHN DOE",
    "token": "1234-5678-9012-3456-7890",
    "units": "45.8 kWh",
    "amount": 5000,
    "disco": "IKEDC",
    "status": "success"
  }
}
```

<Note>
  Always **validate the meter** before purchase. A failed validation means the
  meter number is incorrect or the DISCO service is temporarily unavailable.
</Note>
